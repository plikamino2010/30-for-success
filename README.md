# Wait 30 ⏳ — รอก่อนซื้อ

แอป PWA สำหรับช่วยลดการซื้อของตามอารมณ์ (impulse buying) โดยให้บันทึกของที่อยากได้
ตั้งระยะเวลา "รอคิด" ตามราคาสินค้า แสดงนับถอยหลัง และให้ทำแบบทดสอบทบทวนความต้องการทุก 5 วัน

ข้อมูลทั้งหมดเก็บไว้ใน **localStorage ของเบราว์เซอร์เท่านั้น** ไม่มีเซิร์ฟเวอร์ ไม่มีระบบล็อกอิน

## 2. การติดตั้งและรัน (Install & Run)

ต้องมี **Node.js 18+** ติดตั้งไว้ในเครื่อง

```bash
# 1. เข้าไปที่โฟลเดอร์โปรเจกต์
cd wait30

# 2. ติดตั้ง dependencies (ต้องมีอินเทอร์เน็ต)
npm install

# 3. รันโหมดพัฒนา (dev server)
npm run dev
```

จากนั้นเปิดเบราว์เซอร์ไปที่ URL ที่แสดงในเทอร์มินัล (ปกติคือ `http://localhost:5173`)

> 💡 บนมือถือ: เปิด dev server ด้วย `npm run dev -- --host` แล้วเข้าจาก IP เครื่องคอมพิวเตอร์
> ผ่าน Wi-Fi เดียวกัน เพื่อทดสอบบนหน้าจอมือถือจริง

## 3. การ Build (Production Build)

```bash
npm run build
```

ไฟล์ที่ build แล้วจะอยู่ในโฟลเดอร์ `dist/` — สามารถนำไป deploy บน static hosting
ใดก็ได้ (เช่น Netlify, Vercel, GitHub Pages, Cloudflare Pages) เพราะไม่มี backend

ดูตัวอย่างก่อน deploy จริงด้วย:

```bash
npm run preview
```

### ทำให้เป็น PWA ติดตั้งบนมือถือ
หลัง deploy ขึ้น HTTPS แล้ว เปิดเว็บในเบราว์เซอร์มือถือ แล้วเลือก
"เพิ่มลงในหน้าจอหลัก / Add to Home Screen" — แอปจะทำงานแบบ standalone
และมี service worker cache หน้าเว็บไว้ใช้งานแบบออฟไลน์เบื้องต้น

> หมายเหตุไอคอน: ในโปรเจกต์ใช้ `public/icon.svg` เป็นไอคอนแอป ซึ่งใช้ได้กับเบราว์เซอร์ส่วนใหญ่
> ถ้าต้องการไอคอน PNG ขนาด 192x192 และ 512x512 (สำหรับความเข้ากันได้สูงสุด)
> ให้แปลงจาก `icon.svg` ด้วยโปรแกรมแปลงไฟล์ภาพ แล้วเพิ่ม entry ใน `manifest.json`

## 4. ข้อมูลตัวอย่าง (Mock Data)

ในไฟล์ `src/data/mockData.ts` มีรายการตัวอย่าง 5 รายการ ซึ่งจะถูกเพิ่มให้อัตโนมัติ
**ครั้งแรกที่เปิดแอป** (เมื่อยังไม่มีข้อมูลใน localStorage) เพื่อให้เห็นตัวอย่างทุกสถานะ:

| รายการ | ราคา | สถานะ | หมายเหตุ |
|---|---|---|---|
| หูฟังบลูทูธไร้สาย | 390 | ✅ พร้อมตัดสินใจ | เพิ่มมา 2 วัน, รอ 1 วัน → ครบกำหนดแล้ว |
| รองเท้าวิ่งรุ่นใหม่ | 1,590 | ⏳ กำลังรอ | เพิ่มมา 3 วัน, รอ 7 วัน, ยังไม่ถึงรอบทบทวน |
| แท็บเล็ตสำหรับวาดรูป | 8,900 | ⏳ กำลังรอ | เพิ่มมา 6 วัน, รอ 14 วัน, มีประวัติทบทวนครั้งที่ 1 แล้ว |
| กล้องมิลเลอร์เลสตัวใหม่ | 25,900 | 🛍️ ซื้อแล้ว | ตัวอย่างรายการที่ตัดสินใจซื้อแล้ว |
| กระเป๋าสะพายแบรนด์เนม | 2,490 | 🚫 ยกเลิกแล้ว | ตัวอย่างที่ทบทวนแล้วคะแนนต่ำ เลยยกเลิก |

ถ้าต้องการล้างข้อมูลตัวอย่างทั้งหมดและเริ่มใหม่ ให้เปิด Developer Tools ของเบราว์เซอร์ →
Application/Storage → Local Storage → ลบ key `wait30_items` และ `wait30_seeded`
แล้ว reload หน้าเว็บ (แอปจะเริ่มจากรายการเปล่า ไม่เพิ่มข้อมูลตัวอย่างซ้ำ)

## 6. ข้อมูลถูกเก็บไว้ที่ไหน (Where Data Is Stored)

ข้อมูลทั้งหมด (รายการสินค้า รูปภาพ ประวัติการทบทวน) ถูกเก็บไว้ใน
**`localStorage` ของเบราว์เซอร์บนอุปกรณ์นั้นๆ เท่านั้น** ภายใต้ key:

- `wait30_items` — array ของรายการสินค้าทั้งหมด (JSON)
- `wait30_seeded` — flag บอกว่าเคยเพิ่มข้อมูลตัวอย่างไปแล้วหรือยัง

ผลลัพธ์ของเรื่องนี้:
- **ไม่มีการส่งข้อมูลไปเซิร์ฟเวอร์ใดๆ** ทุกอย่างทำงานในเบราว์เซอร์
- ข้อมูลจะ**ไม่ sync ข้ามอุปกรณ์/เบราว์เซอร์** — ถ้าเปิดในเบราว์เซอร์อื่นหรืออุปกรณ์อื่น
  จะเห็นเป็นแอปเปล่า (พร้อมข้อมูลตัวอย่างใหม่)
- ถ้าล้าง "ข้อมูลเว็บไซต์ (site data)" ของเบราว์เซอร์ ข้อมูลทั้งหมดจะหายไป
- รูปภาพที่อัปโหลดจะถูกย่อขนาดและแปลงเป็น base64 ก่อนเก็บ เพื่อไม่ให้กิน
  พื้นที่ localStorage มากเกินไป (localStorage มักมีขีดจำกัดประมาณ 5-10MB ต่อโดเมน)
- ถ้า localStorage เต็ม (มักเกิดจากรูปภาพจำนวนมาก) แอปจะ**ไม่บันทึกข้อมูลล่าสุดแบบเงียบๆ**
  อีกต่อไป — จะมีแบนเนอร์สีแดงแจ้งเตือนที่ด้านบนของแอปให้ทราบ พร้อมคำแนะนำให้ลบรูปภาพ/รายการที่ไม่ใช้
- ข้อมูลที่โหลดจาก localStorage จะถูกตรวจสอบและเติมค่าเริ่มต้นให้ฟิลด์ที่หายไป/ผิดรูปแบบ
  เพื่อป้องกันแอปพังหากข้อมูลถูกแก้ไขจากภายนอกหรือมาจากเวอร์ชันเก่า

## 7. กฎการคำนวณ (สำหรับอ้างอิง)

```
suggestWaitDays(price):
  price < 500        -> 1 วัน
  500 <= price < 3000 -> 7 วัน
  3000 <= price < 10000 -> 14 วัน
  price >= 10000     -> 30 วัน

decisionDate = startDate + waitDays
daysRemaining = decisionDate - today
ถ้า daysRemaining <= 0 -> สถานะ = "ready" (พร้อมตัดสินใจ)

รอบทบทวน (review checkpoints): ทุก 5 วัน นับจากวันที่เพิ่มรายการ (createdAt)
  เช่น ครั้งที่ 1 = วันที่ 5, ครั้งที่ 2 = วันที่ 10, ครั้งที่ 3 = วันที่ 15, ...
  ทำต่อเนื่องไปเรื่อยๆ ไม่มีจำกัดที่ 30 วัน (รองรับกรณีตั้งจำนวนวันรอเองมากกว่า 30)

คะแนนรวม quiz = q1+q2+q3+q4+q5 (แต่ละข้อ 0-5, รวม 0-25)
  0-9   -> ความอยากลดลงแล้ว พิจารณายกเลิกหรือรอเพิ่ม
  10-17 -> ยังไม่แน่ใจ รอเพิ่มอีก 5-10 วัน
  18-25 -> ความอยากยังสูง ซื้อได้ถ้าไม่กระทบงบประมาณ
```
